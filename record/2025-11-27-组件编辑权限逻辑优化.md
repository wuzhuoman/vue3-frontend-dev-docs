# 组件编辑权限逻辑优化记录

**日期**: 2025-11-27  
**修改人**: AI编码助手  
**涉及模块**: Vue3前端组件权限控制  

## 问题描述

在 `CavdComponentDetail.vue` 组件中，编辑权限的判断逻辑存在问题：

1. 原实现使用的是从父组件传递的漏洞的 `source` 字段
2. 而实际上应该基于组件自己的 `source` 字段来判断是否允许编辑
3. 这导致某些场景下权限判断不准确，例如：
   - 漏洞来源是 `'catarc'`，但组件来源是 `'scantist'` 时
   - 系统错误地允许编辑不应编辑的组件

## 涉及文件

| 文件路径 | 修改类型 | 修改内容 |
|---------|---------|---------|
| `src/views/setting/components/CavdComponentDetail.vue` | 功能修改 | 修改权限判断逻辑，基于组件的 `source` 而非漏洞的 `source` |

## API 调用分析

组件详情 API:
- **接口**: `GET /hub/management/libraries/{library_id}/`
- **返回数据示例**:
```json
{
    "library_id": 13165,
    "name": "hp-ux",
    "language": "closed source",
    "platform": "Closed Source",
    "vendor": "hp",
    "source": "scantist",  // 这个字段才是正确的判断依据
    "description": null
}
```

## 修改内容详解

### 1. 添加组件级 `source` 变量

```javascript
// 使用组件自己的source值，而不是从父组件传递的漏洞的source值
const componentSource = ref(props.source); // 初始使用props.source，API返回后更新
```

### 2. 在API调用后更新 `componentSource`

```javascript
const getLibraries = () => {
  return orgStore.getLibraries({ library_id: props.curId }).then((res) => {
    ruleForm.library_name = res.name || "";
    ruleForm.vendor = res.vendor || "";
    ruleForm.platform =
      res.platform == "NOT_SPECIFIED" ? "" : res.platform || "";
    ruleForm.description = res.description || "";
    
    // 更新组件source值
    componentSource.value = res.source || "";
  });
};
```

### 3. 修改模板中的判断条件

将两处 `v-if="source == 'catarc'"` 改为 `v-if="componentSource == 'catarc'"`：

1. **删除版本按钮的判断**:
```html
<el-button
  v-if="componentSource == 'catarc'"
  type="primary"
  class="m-opt_btn"
  :icon="DeleteFilled"
  color="var(--el-color-danger)"
  @click="handleDelClick(row)"
/>
```

2. **整个操作按钮组的显示判断**:
```html
<el-row
  class="m-btn"
  v-if="componentSource == 'catarc'"
  justify="end"
  align="middle"
>
```

## 数据流对比

### 修改前
```
后端API(漏洞详情) → selectedRow.source → CavdComponentList → CavdComponentDetail → 判断编辑权限
```

### 修改后
```
后端API(组件详情) → componentSource → 直接在CavdComponentDetail中判断编辑权限
```

## 实现效果

1. **权限控制更加精确**：
   - 只有当组件自身的 `source` 为 `'catarc'` 时才显示编辑和删除按钮
   - 不再受漏洞来源影响

2. **用户体验更加合理**：
   - 用户只能编辑真正允许编辑的组件
   - 避免了权限混乱的情况

3. **系统安全性提升**：
   - 防止了误操作不应编辑的组件数据
   - 增强了数据完整性保护
