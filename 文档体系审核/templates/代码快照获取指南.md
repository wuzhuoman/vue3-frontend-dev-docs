# 代码快照信息获取指南

## 📋 需要获取的信息

每次审核开始前，必须获取以下Git信息并填入`审核报告.md`：

| 信息项 | 示例 | 获取方法 |
|--------|------|----------|
| Git分支 | rc-v4.19.0 | `git branch --show-current` |
| 最新提交 | a1b2c3d4 fix: 修复用户登录问题 | `git log -1 --format="%H %s"` |
| 提交时间 | 2025-12-15 14:30:25 +0800 | `git log -1 --format="%ai"` |
| 项目版本 | 4.19.0 | 从`package.json`读取`version`字段 |
| 分支状态 | Clean | `git status --porcelain` |

## 🔧 获取命令详解

### 1. 获取当前Git分支

```bash
git branch --show-current
```

**输出示例**：
```
rc-v4.19.0
```

**说明**：显示当前所在的分支名称

---

### 2. 获取最新提交信息

```bash
git log -1 --format="%H %s"
```

**输出示例**：
```
a1b2c3d4e5f6789012345678901234567890abcd fix: 修复用户登录问题
```

**说明**：
- `%H`：完整的commit hash（40个字符）
- `%s`：提交消息
- 建议在报告中使用前8位（a1b2c3d4），便于阅读

**可选命令**：只获取commit hash的前8位
```bash
git log -1 --format="%h"
# 输出：a1b2c3d4
```

---

### 3. 获取提交时间

```bash
git log -1 --format="%ai"
```

**输出示例**：
```
2025-12-15 14:30:25 +0800
```

**说明**：
- `%ai`：ISO 8601格式的提交时间
- 包含日期、时间和时区

---

### 4. 检查分支状态

```bash
git status --porcelain
```

**输出示例1（Clean状态）**：
```
(无输出)
```

**输出示例2（Dirty状态）**：
```
M src/api/user.ts
A src/components/NewComponent.vue
D src/utils/old-function.js
```

**说明**：
- 无输出：分支状态为Clean（无未提交更改）
- 有输出：分支状态为Dirty（有未提交更改）
  - `M`：已修改
  - `A`：已添加
  - `D`：已删除

**重要提示**：如果分支状态为Dirty，建议先提交代码再进行审核，确保审核的准确性和可追溯性。

---

### 5. 获取项目版本

**方法1：直接读取package.json**
```bash
# 在项目根目录执行
cat package.json | grep "\"version\""
```

**输出示例**：
```json
  "version": "4.19.0",
```

**方法2：使用jq工具（如果已安装）**
```bash
cat package.json | jq -r '.version'
```

**输出示例**：
```
4.19.0
```

**说明**：获取`package.json`文件中的`version`字段值

---

## 📝 完整获取流程

### 自动化脚本（推荐）

在项目根目录执行以下命令，一次性获取所有信息：

```bash
#!/bin/bash

echo "=== Git快照信息 ==="
echo "Git分支: $(git branch --show-current)"
echo "最新提交: $(git log -1 --format="%h %s")"
echo "提交时间: $(git log -1 --format="%ai")"
echo "项目版本: $(cat package.json | grep "\"version\"" | sed 's/.*"version".*: *"\(.*\)".*/\1/')"

# 检查分支状态
if [ -z "$(git status --porcelain)" ]; then
  echo "分支状态: Clean"
else
  echo "分支状态: Dirty (有未提交的更改)"
  echo "未提交的文件:"
  git status --porcelain
fi
```

**输出示例**：
```
=== Git快照信息 ===
Git分支: rc-v4.19.0
最新提交: a1b2c3d4 fix: 修复用户登录问题
提交时间: 2025-12-15 14:30:25 +0800
项目版本: 4.19.0
分支状态: Clean
```

## ✅ 填写到审核报告

将获取的信息填入`审核报告.md`的"代码快照信息"表格：

```markdown
## 📊 代码快照信息

| 信息项 | 内容 |
|--------|------|
| Git分支 | rc-v4.19.0 |
| 最新提交 | a1b2c3d4 fix: 修复用户登录问题 |
| 提交时间 | 2025-12-15 14:30:25 +0800 |
| 项目版本 | 4.19.0 |
| 分支状态 | Clean |
```

## ⚠️ 注意事项

1. **分支状态为Dirty时**：
   - 建议先提交代码再进行审核
   - 如果必须审核，请在报告中注明"Dirty状态，有X个未提交的文件"

2. **多个开发者协作**：
   - 确保本地代码是最新的（`git pull`）
   - 记录审核时的具体commit hash，便于追溯

3. **版本匹配**：
   - 确保`package.json`中的version与Git tag或分支名称一致
   - 如有不一致，请在报告中注明

4. **审核过程中的代码变更**：
   - 审核期间应避免拉取新代码
   - 如果必须更新，请重新获取Git快照信息并更新报告

## 📞 常见问题

**Q1：为什么需要记录Git信息？**
A1：
- 多开发者协作项目，代码持续演进
- 明确审核时对应的代码版本
- 便于后续追溯问题，确认文档与代码的对应关系
- 当代码更新后，可以根据Git信息判断是否需要重新审核文档

**Q2：如果分支状态为Dirty怎么办？**
A2：建议先提交代码再进行审核。如果必须审核，请在报告中注明"Dirty状态"，并列出未提交的文件。

**Q3：如何验证审核报告中的Git信息是否正确？**
A3：可以使用以下命令验证：
```bash
# 验证分支
git branch --show-current

# 验证提交
git log -1 --oneline

# 验证版本
cat package.json | grep version
```

**Q4：审核期间代码被更新了怎么办？**
A4：
- 如果只是其他分支更新，不影响审核
- 如果当前分支被更新（`git pull`），建议重新获取Git快照信息并更新报告
- 审核过程中应避免执行`git pull`，确保审核的一致性
